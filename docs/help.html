RISQUE help

Quick reference for this app — read before you log in or share with new players.

Login screen

  • Player rows — Enter a name, then click a color swatch to assign that color. Each name and each color must be unique.
  • Empty start — Name fields are blank when you open login. The map stats table also shows no roster until you log in. Use a Preset button if you want a saved lineup in one click.
  • Save-folder hint (host) — If you start from the Windows launcher / local disk helper, you may see a small green badge on login confirming the save API is reachable; it goes away once the game begins.
  • Game manual — Full rules overview (Risk-style).
  • Help — This page.
  • Log in — Starts a new game with the players you entered (after setup phases).
  • Load game — Choose a saved .json game file to restore a previous session.

Where the game runs

Most play happens in game.html (runtime shell): the map, phase panels, and buttons for attack, deploy, income, card play, etc. The exact screen depends on the current phase saved in your game state.

Host map — corner controls

On the host map (not the public TV tab), a toolbar sits on the map graphic:

  • Top row — NEW GAME, LOAD GAME, SAVE GAME, GRACE, PUBLIC (TV window), HIDE BUTTONS.
  • HIDE BUTTONS — Collapses the top row to a single SHOW BUTTONS control in the same grid slot (same size as before). On the bottom row it hides REPLAY and SAVE + REPLAY only; MANUAL, HELP, the Autosave chip, EMERGENCY, and the round badge stay visible.
  • Bottom row — MANUAL / HELP (these docs), REPLAY (Wayback), SAVE + REPLAY (same export as SAVE), the Autosave Config chip, EMERGENCY snapshot, and the large ROUND indicator.

Autosave & replay policy

Click Autosave Config on the host to set how often checkpoints hit your connected save folder. The default is 2 · Safe with lean fun. Four options:

  1. Safe & fun (full replay) — Every player turn: game checkpoint plus granular replay segments on disk (DD.json, rNpM.json, etc.). Heaviest on disk; best if you want Wayback to rebuild from files alone.
  2. Safe with lean fun — Turn game checkpoints; granular replay files only when the host turns that path on (e.g. manual save, game win, opening Wayback). Fewer small replay files between exports.
  3. Safe, no replay disk — Game checkpoints only (e.g. rNpMgame.json style turns). No DD.json or per-turn rNpM.json replay tapes. In a finished session you can still use in-memory tape for postgame Wayback until you close the tab.
  4. No autosave — No automatic turn/round exports; you persist when you choose SAVE, SAVE + REPLAY, or EMERGENCY.

SAVE GAME and SAVE + REPLAY always write the current game JSON (replay stripped from the main file) and a full-session replay JSON from the deal through “now”, even in tier 4 — you do not need prior replay files on disk for that export.

Policy is stored per browser (localStorage) and copied into new games from login. Use Choose save folder… in the same dialog if the browser asks you to grant a directory (HTTPS/localhost). The RISQUE.bat launcher can talk to a small local API so saves land under the usual Windows save path with fewer prompts.

Map & attack phase (typical)

  • Click one of your territories with at least 2 armies, then an adjacent enemy territory (or valid aerial link if enabled).
  • Use Roll, Blitz, or other attack buttons on the panel.
  • After capturing, you will be prompted to move armies into the conquered territory (at least the minimum required).
  • Campaign instant — If an automated campaign run eliminates a player, the chain stops so you can handle elimination (cards, troops, follow-up phases) instead of continuing across the whole map in one go.
  • Reinforcement phase moves you to placing armies between your territories before drawing cards / ending the turn, when the flow calls for it.

Saves & browser data

  • The active game is kept in the browser’s localStorage under the key used by the app (e.g. when you continue from the runtime).
  • Save game / Save + replay on the map suggests a basename like RISQUE 2026-04-17 8:11AM (two spaces after RISQUE and before the time). You normally get two files: the game JSON and a -replay.json sibling with the full session tape. The colon in the time becomes a hyphen on Windows (e.g. 8-11AM). With http://localhost, HTTPS, or the launcher disk API, files can go straight into your chosen save folder; pure file:// often lands in Downloads.
  • Emergency — Forces an immediate game + replay snapshot (similar idea to SAVE) when you need a mid-round rescue.
  • Load game on login replaces that stored state from your JSON file.
  • Clearing site data for this origin will erase in-progress games unless you have exported a JSON backup.

Privacy gate

When the app shows a privacy step, it is so the next player does not see secret information (hands, draws, etc.). Pass the device or switch display as your group prefers.

Wayback / instant replay

The REPLAY button on the host map opens Wayback (the replay machine) in the control panel — no pop-up. It plays the session tape: deal, deploy, battles, eliminations, as map snapshots (not a full dice or card animation).

  • Start options — Full run from the territory deal (default), skip to the first battle, or (when the tape has round stamps) jump to the first frame at or after a chosen round.
  • During playback — Look for SKIP TO END / CANCEL; Esc cancels.
  • Sources — Wayback prefers the in-memory tape from the current tab, then localStorage bootstrap, then files in your connected save folder (e.g. merged DD.json + rNpM.json when those exist).
  • End of game — After a normal finish, the app can open replay automatically; you should still see the full session from the deal if the tab stayed open, even when autosave wrote no granular replay files.
  • Public TV — A mirror tab can follow the same frames when host mirroring already works for live play.

Replay is authored on the host. If playback looks short, the folder may only have a partial export (e.g. deal file only) and the in-memory tape may have been lost (reload/crash). Use SAVE + REPLAY before closing for a portable full tape. Older saves may lack round stamps on the tape, which limits “start from round” choices.

Public monitor — cardplay recap

During committed cardplay on the public display, when a wildcard is used as a territory card, the monitor shows the chosen territory name prominently; a second line beneath indicates the effect at a glance (for example +2, −2, or ACQUIRED), similar to normal territory cards.

If something looks wrong

  • Combat dice or income once looked wrong — Save, refresh, and continue; if it persists, load a backup JSON. Odd totals can happen if the browser had a stale tab or interrupted save.
  • Refresh the page; if the save is corrupt, try Load game from a known-good JSON file.
  • Use the launcher / tools menu from index.html?menu=1 if your project includes it (clear logs, test setups).
  • For a fresh start, use login again with new players (or load an earlier save).
Tip: Open Manual and Help in a separate tab from login so you can read while keeping the login form open.
